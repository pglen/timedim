   As mentioned in the intruductory pages, TimeDim will not create a bootable disk upon copy, so it cannot be used as a substitution for drive creation tools. However, drivecopy will create an exact duplicate of any data, as long as it is accessable from the machine it is running on.

 TimeDim will copy network shares, removable drives, Samba (UNIX) shares, Samba. TimeDim will operate on USB, FireWire, SATA, ATA drives with no distinction to the drive type.

 The TimeDim log facility is invaluable tool for auditing the suuccess of the backups. Any error, inability to copy and major events (ex: application start/stop) is logged into timedim's log facility.

 Because timedim tests for file modification dates, copy sessions can be started/stopped at will, timedim will resume copying at the correct place. Timedim  date compare is file atomic.
